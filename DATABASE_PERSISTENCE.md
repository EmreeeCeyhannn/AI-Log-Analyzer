# Database Persistence Implementation

## Overview

The LogBatcher cache tree system now includes comprehensive database persistence. All cache entries, template statistics, and tree snapshots are automatically saved to SQLite and restored on app restart.

## Features Implemented

### 1. **Persistent Cache Storage** (`log_parser/pipeline/db.py`)

- **SQLite Database**: Stores all cache and template data in `log_parser/data/templates.db`
- **Thread-safe Operations**: Uses locks for concurrent access safety
- **Automatic Initialization**: Schema is created on first run

### 2. **Database Schema**

Four main tables:

#### `cache_entries`

- Stores template cache entries with SHA256 keys
- Tracks hit count for hot cache optimization
- Auto-updates with timestamp tracking

#### `template_stats`

- Global template usage statistics
- Tracks cache vs SLM generation ratio
- Records first_seen and last_used timestamps
- Allows recovery of template patterns across restarts

#### `tree_history`

- Snapshots of the complete tree state after each parse
- Stores full tree JSON with batch metadata
- Enables quick recovery of last known good state

#### `example_logs`

- Individual log examples for each template
- Links to template_stats for relationship management
- Tracks log level, event type, and source (CACHE/SLM)

### 3. **Cache Enhancement** (Updated `log_parser/pipeline/cache.py`)

```python
class TemplateCache:
    def __init__(self, db_path: str | Path | None = None):
        # Pre-loads all cache entries from database on startup
        self._load_from_db()
```

- **Dual-layer Caching**: In-memory + persistent storage
- **Auto-save on Set**: Every cache write persists to database
- **Pre-load on Startup**: Recovers all previous cache entries

### 4. **Registry Enhancement** (Updated `log_parser/pipeline/intelligence.py`)

```python
class TemplateIntelligenceRegistry:
    def set_db(self, db: TemplateDatabase):
        # Loads historical template stats
        self._load_from_db()
```

- **Template Recovery**: Loads all historical template stats on startup
- **Auto-persistence**: Every `record()` call saves to database
- **Usage Tracking**: Maintains cache vs SLM statistics over time

### 5. **Factory Integration** (Updated `log_parser/factory.py`)

```python
def build_pipeline(config_path):
    db = TemplateDatabase(db_path="log_parser/data/templates.db")
    cache = TemplateCache(db_path="log_parser/data/templates.db")

- Single source of truth for database initialization
- Ensures all components share the same database instance
- Automatic data sync across components

## How It Works

### On First Run:

1. Database is created with schema
2. Cache is empty (no prior entries)
3. Templates are parsed and saved as they're generated
4. Tree snapshots are saved after each parse
5. Cache statistics become available

### On Subsequent Runs:

1. Database loads all previous cache entries into memory
2. Registry loads all historical template statistics
3. UI displays "Load Previous Tree" button with data
4. Cache hits significantly improve performance (SLM reuse)
5. Statistics show cumulative data across all runs

### After Each Parse:

1. Pipeline processes logs normally
2. Cache hits are logged (faster, reuses SLM output)
3. New templates are generated by SLM and cached
4. Tree snapshot is saved to database
5. Template statistics are updated in database

## Data Recovery Scenarios

| Scenario           | Recovery Method            | Result                      |
| ------------------ | -------------------------- | --------------------------- |
| App restart        | Automatic on startup       | All cache + stats loaded    |
| Browser refresh    | Click "Load Previous Tree" | Last tree snapshot restored |
| Manual recovery    | Access database directly   | Full audit trail available  |
| performance tuning | Analyze cache statistics   | Identify hot templates      |

```

## Usage Examples

### View Previous Trees

```python
from log_parser.pipeline.db import TemplateDatabase

db = TemplateDatabase()
tree = db.get_latest_tree_snapshot()  # Last saved tree
```

### Retrieve Template Statistics

```python
stats = db.get_template_stats()  # All cached templates
for stat in stats:
    print(f"{stat['template_text']}: {stat['usage_count']} uses")
```

### Clear Old Cache

```python
deleted = db.clear_old_entries(days=30)  # Remove 30+ day old entries
print(f"Cleaned {deleted} old cache entries")
```

## Performance Benefits

1. **Cache Hit Speed**: Retrieved SLM outputs without re-processing
2. **Faster Parsing**: Similar log patterns reuse cached templates
3. **Reduced Latency**: Eliminates redundant SLM model calls
4. **Memory Efficiency**: Hot templates served from cache
5. **Pattern Recognition**: Over time, system learns common patterns

## Thread Safety

All database operations use `RLock` (reentrant locks) to ensure:

- Concurrent read access
- Safe write operations
- No race conditions
- Atomic updates
